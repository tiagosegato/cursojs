Exercícios Práticos

Pratique seus conhecimentos com exercícios específicos para cada módulo do curso

Exercícios por Módulo

Abaixo você encontrará exercícios práticos para cada módulo do curso. Estes exercícios foram cuidadosamente elaborados para reforçar os conceitos aprendidos e ajudar no desenvolvimento das habilidades necessárias para se tornar um desenvolvedor backend proficiente.

Recomendamos que você tente resolver os exercícios por conta própria antes de verificar as soluções. A prática é fundamental para o aprendizado efetivo de programação.

Módulo 1: Lógica de Programação com JavaScript

Exercício 1: Calculadora de Preço de Produtos

Crie um programa em JavaScript que:

  1. Declare um array de objetos representando produtos com as propriedades: código, descrição, preço de custo, margem de lucro (em percentual)
  2. Implemente uma função que calcule o preço de venda de cada produto (preço de custo + margem de lucro)
  3. Exiba uma lista formatada com todos os produtos, mostrando código, descrição, preço de custo, margem de lucro e preço de venda
  4. Calcule e exiba o valor total do estoque (soma dos preços de venda de todos os produtos)

Exercício 2: Filtro de Produtos

Desenvolva um programa que:

  1. Utilize o array de produtos do exercício anterior
  2. Implemente funções para filtrar produtos por:
    • Faixa de preço (mínimo e máximo)
    • Parte da descrição (busca por texto)
    • Margem de lucro acima de um determinado valor
  3. Crie uma função que ordene os produtos por preço (crescente ou decrescente)
  4. Exiba os resultados de cada filtro e ordenação

Exercício 3: Controle de Estoque

Crie um sistema simples de controle de estoque que:

  1. Adicione a propriedade "quantidade em estoque" aos objetos de produto
  2. Implemente funções para:
    • Adicionar quantidade a um produto específico
    • Remover quantidade (venda) de um produto específico
    • Verificar se há estoque suficiente antes de realizar uma venda
    • Alertar quando o estoque estiver abaixo de um valor mínimo definido
  3. Simule algumas operações de entrada e saída de produtos
  4. Exiba um relatório final do estoque
Módulo 2: Git, GitHub e GitLab

Exercício 1: Configuração e Primeiro Repositório

  1. Configure o Git em sua máquina local com seu nome e email
  2. Crie um novo repositório local para o projeto "Sistema de Vendas"
  3. Adicione um arquivo README.md com a descrição do projeto
  4. Realize o primeiro commit
  5. Crie um repositório no GitHub ou GitLab
  6. Conecte seu repositório local ao remoto
  7. Faça o push do seu commit para o repositório remoto

Exercício 2: Branches e Merge

  1. No repositório do exercício anterior, crie uma branch chamada "feature/produtos"
  2. Na nova branch, adicione um arquivo produtos.js com o código de um dos exercícios do Módulo 1
  3. Faça commit das alterações
  4. Volte para a branch main/master
  5. Crie um arquivo .gitignore e adicione configurações básicas
  6. Faça commit das alterações
  7. Realize o merge da branch "feature/produtos" na main/master
  8. Resolva qualquer conflito que possa surgir
  9. Faça push de todas as alterações para o repositório remoto

Exercício 3: Colaboração e Pull Requests

  1. Forme uma dupla com outro aluno do curso
  2. Faça um fork do repositório do seu colega
  3. Clone o repositório forkado para sua máquina
  4. Crie uma branch para implementar uma nova funcionalidade
  5. Implemente a funcionalidade (por exemplo, adicionar um arquivo clientes.js)
  6. Faça commit e push para o seu fork
  7. Crie um Pull Request para o repositório original
  8. Seu colega deve revisar o PR, sugerir alterações se necessário, e fazer o merge
  9. Repita o processo invertendo os papéis
Módulo 3: JavaScript e Orientação a Objetos

Exercício 1: Classes de Produtos

  1. Crie uma classe Produto com as seguintes propriedades:
    • código
    • descrição
    • unidade
    • preço de custo
    • margem de lucro
    • estoque atual
    • estoque mínimo
  2. Implemente os seguintes métodos:
    • calcularPrecoVenda(): retorna o preço de venda baseado no custo e margem
    • adicionarEstoque(quantidade): adiciona quantidade ao estoque atual
    • removerEstoque(quantidade): remove quantidade do estoque (com verificação)
    • verificarEstoqueBaixo(): retorna true se estoque atual < estoque mínimo
    • toString(): retorna uma representação textual do produto
  3. Crie alguns objetos de produto e teste todos os métodos

Exercício 2: Herança e Polimorfismo

  1. Crie uma classe ProdutoPerecivel que herda de Produto, adicionando:
    • data de validade
    • método para verificar se está vencido
    • sobrescreva o método toString() para incluir a data de validade
  2. Crie uma classe ProdutoEletronico que herda de Produto, adicionando:
    • voltagem
    • garantia (em meses)
    • sobrescreva o método toString() para incluir as novas propriedades
  3. Crie uma classe Estoque que gerencia uma coleção de produtos (de qualquer tipo)
  4. Implemente métodos para adicionar produtos, listar todos, filtrar por tipo, etc.
  5. Teste o polimorfismo criando um array com diferentes tipos de produtos

Exercício 3: Encapsulamento e Métodos Estáticos

  1. Modifique a classe Produto para usar encapsulamento:
    • Torne as propriedades privadas (usando # ou convenção _)
    • Crie getters e setters para cada propriedade
    • Adicione validações nos setters (ex: preço não pode ser negativo)
  2. Adicione métodos estáticos à classe Produto:
    • compararPorPreco(produtoA, produtoB): para ordenação
    • calcularValorTotal(produtos): soma o valor de venda de uma lista de produtos
    • filtrarPorCategoria(produtos, categoria): filtra produtos por categoria
  3. Teste os novos métodos e o encapsulamento
Módulo 4: Padrões de Desenvolvimento de Software

Exercício 1: Implementação do Padrão Singleton

  1. Crie uma classe GerenciadorProdutos que implemente o padrão Singleton
  2. A classe deve manter uma lista de produtos
  3. Implemente métodos para adicionar, remover, buscar e listar produtos
  4. Teste o Singleton criando múltiplas "instâncias" e verificando que todas apontam para o mesmo objeto

Exercício 2: Implementação do Padrão Factory

  1. Crie uma Factory de Produtos que possa criar diferentes tipos de produtos:
    • ProdutoSimples
    • ProdutoPerecivel
    • ProdutoEletronico
  2. A factory deve ter um método createProduto(tipo, dados) que retorna o tipo apropriado de produto
  3. Teste a factory criando diferentes tipos de produtos

Exercício 3: Implementação do Padrão Observer

  1. Implemente o padrão Observer para monitorar mudanças no estoque de produtos
  2. Crie uma classe EstoqueSubject que mantém uma lista de observadores
  3. Crie interfaces/classes para diferentes tipos de observadores:
    • AlertaEstoqueBaixo: notifica quando o estoque está abaixo do mínimo
    • RegistroLog: registra todas as operações de estoque
    • NotificacaoCompras: gera uma notificação para o setor de compras quando necessário
  4. Implemente a lógica para notificar os observadores quando houver mudanças no estoque
  5. Teste o sistema realizando operações de adição e remoção de estoque
Módulo 5: Introdução a WebServices

Exercício 1: Consumo de API REST

  1. Utilize a API pública JSONPlaceholder (https://jsonplaceholder.typicode.com/)
  2. Crie funções para:
    • Obter todos os posts
    • Obter um post específico por ID
    • Obter comentários de um post específico
    • Criar um novo post
    • Atualizar um post existente
    • Excluir um post
  3. Utilize fetch ou axios para fazer as requisições
  4. Implemente tratamento de erros adequado
  5. Exiba os resultados no console ou em uma página HTML simples

Exercício 2: Simulação de API de Produtos

  1. Crie um arquivo JSON com dados de produtos (pelo menos 10 produtos)
  2. Utilize o json-server para criar uma API REST local: npm install -g json-server json-server --watch produtos.json
  3. Crie um cliente JavaScript que consuma esta API, com funções para:
    • Listar todos os produtos
    • Buscar produto por ID
    • Filtrar produtos por preço ou categoria
    • Adicionar novo produto
    • Atualizar produto existente
    • Excluir produto
  4. Crie uma interface HTML simples para interagir com a API

Exercício 3: Integração com API Externa

  1. Escolha uma API pública que forneça dados relevantes para um sistema de vendas (ex: API de CEP, cotação de moedas, etc.)
  2. Crie funções para consumir os endpoints relevantes da API escolhida
  3. Integre esses dados com o sistema de produtos:
    • Por exemplo, se for uma API de CEP, use-a para validar endereços de clientes
    • Se for uma API de cotação, use-a para converter preços para diferentes moedas
  4. Implemente cache de resultados para evitar chamadas desnecessárias à API
  5. Crie uma interface simples para demonstrar a integração
Módulo 6: Node.js

Exercício 1: Servidor HTTP Básico

  1. Crie um servidor HTTP básico com Node.js nativo (sem frameworks)
  2. O servidor deve responder a diferentes rotas:
    • GET /: Página inicial com informações sobre a API
    • GET /produtos: Retorna lista de produtos (de um array em memória)
    • GET /produtos/:id: Retorna um produto específico
    • POST /produtos: Adiciona um novo produto
    • PUT /produtos/:id: Atualiza um produto existente
    • DELETE /produtos/:id: Remove um produto
  3. Implemente o parsing de JSON para as requisições POST e PUT
  4. Teste o servidor usando Postman ou curl

Exercício 2: API REST com Express

  1. Refatore o exercício anterior usando Express.js
  2. Organize o código seguindo o padrão MVC:
    • Models: Lógica de negócios e dados
    • Controllers: Manipulação das requisições
    • Routes: Definição das rotas
  3. Adicione validação de dados nas rotas POST e PUT
  4. Implemente middleware para logging de requisições
  5. Adicione tratamento de erros global
  6. Implemente paginação na rota GET /produtos
  7. Adicione filtros (ex: por preço, categoria) na rota GET /produtos
  8. Teste a API usando Postman ou curl

Exercício 3: Sistema de Vendas Simplificado

  1. Crie uma API para um sistema de vendas simplificado com Express
  2. Implemente as seguintes entidades e rotas:
    • Produtos: CRUD completo
    • Clientes: CRUD completo
    • Vendas: Criar venda, listar vendas, obter detalhes de uma venda
  3. Ao criar uma venda:
    • Verifique se o cliente existe
    • Verifique se os produtos existem e têm estoque suficiente
    • Atualize o estoque dos produtos vendidos
    • Calcule o valor total da venda
  4. Implemente relacionamentos entre as entidades
  5. Adicione autenticação básica para proteger as rotas
  6. Documente a API usando comentários ou uma ferramenta como Swagger
  7. Teste a API completa
Módulo 7: Noções de Banco de Dados

Exercício 1: Modelagem de Banco de Dados

  1. Crie um modelo de banco de dados para o sistema de vendas com as seguintes entidades:
    • Produtos
    • Categorias
    • Clientes
    • Vendas
    • ItensVenda
  2. Defina os atributos de cada entidade
  3. Estabeleça os relacionamentos entre as entidades
  4. Crie um diagrama ER (pode usar ferramentas como draw.io, MySQL Workbench, etc.)
  5. Escreva os comandos SQL para criar as tabelas com chaves primárias, estrangeiras e restrições

Exercício 2: Operações SQL Básicas

  1. Utilizando o modelo criado no exercício anterior, escreva comandos SQL para:
    • Inserir pelo menos 5 categorias
    • Inserir pelo menos 10 produtos distribuídos nas categorias
    • Inserir pelo menos 5 clientes
    • Inserir pelo menos 3 vendas com múltiplos itens
  2. Escreva consultas SQL para:
    • Listar todos os produtos com suas categorias
    • Encontrar os produtos mais vendidos
    • Calcular o valor total de vendas por cliente
    • Listar vendas em um período específico
    • Encontrar produtos com estoque abaixo do mínimo
  3. Escreva comandos para atualizar:
    • O preço de todos os produtos de uma categoria específica
    • O endereço de um cliente
    • O status de uma venda
  4. Escreva comandos para excluir:
    • Um produto que nunca foi vendido
    • Uma categoria vazia

Exercício 3: Integração com Node.js

  1. Configure um banco de dados MySQL ou PostgreSQL local
  2. Execute os scripts SQL dos exercícios anteriores para criar e popular o banco
  3. Crie uma aplicação Node.js com Express que se conecte ao banco de dados
  4. Implemente um modelo para cada entidade usando um ORM (Sequelize, TypeORM) ou queries SQL diretas
  5. Crie rotas para:
    • CRUD de produtos
    • CRUD de clientes
    • Criação e consulta de vendas
    • Relatórios (produtos mais vendidos, vendas por período, etc.)
  6. Implemente transações para operações que afetam múltiplas tabelas (ex: criar venda)
  7. Adicione validações e tratamento de erros
  8. Teste a aplicação completa
Módulo 8: Metodologias Ágeis e Scrum

Exercício 1: Planejamento de Sprint

  1. Forme uma equipe de 3-5 pessoas
  2. Escolha um projeto para trabalhar (pode ser o Sistema de Vendas desenvolvido nos exercícios anteriores)
  3. Crie um Product Backlog com pelo menos 15 itens (user stories)
  4. Priorize o Product Backlog
  5. Realize uma sessão de Planning Poker para estimar o esforço de cada item
  6. Planeje um Sprint de 2 semanas, selecionando itens do Product Backlog
  7. Crie um Sprint Backlog detalhado com tarefas para cada item selecionado
  8. Documente todo o processo e as decisões tomadas

Exercício 2: Simulação de Scrum

  1. Com a mesma equipe e projeto do exercício anterior:
  2. Defina os papéis de Scrum (Scrum Master, Product Owner, Desenvolvedores)
  3. Crie um quadro Kanban (pode ser físico ou digital usando Trello, Jira, etc.)
  4. Simule um Sprint de 1 semana:
    • Realize Daily Scrums diários (pode ser por mensagem)
    • Atualize o quadro Kanban conforme as tarefas progridem
    • Documente impedimentos e como foram resolvidos
    • Ao final, realize uma Sprint Review e uma Retrospectiva
  5. Documente as lições aprendidas e como o processo poderia ser melhorado

Exercício 3: Implementação de Práticas Ágeis

  1. Escolha um projeto existente (pode ser o Sistema de Vendas)
  2. Implemente as seguintes práticas ágeis:
    • Integração Contínua: Configure um pipeline básico de CI (pode usar GitHub Actions, Travis CI, etc.)
    • Testes Automatizados: Escreva testes unitários para pelo menos 3 componentes do sistema
    • Revisão de Código: Estabeleça um processo de code review usando Pull Requests
    • Refatoração: Identifique e refatore pelo menos 2 partes do código que poderiam ser melhoradas
  3. Documente o processo de implementação de cada prática
  4. Reflita sobre os benefícios e desafios encontrados
  5. Proponha melhorias para o futuro
