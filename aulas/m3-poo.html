Introdução à Programação Orientada a Objetos

A Programação Orientada a Objetos (POO) é um paradigma que revolucionou o desenvolvimento de software:

  • O que é: Paradigma que utiliza "objetos" para modelar dados e comportamentos
  • JavaScript e POO: Suporte completo a partir do ES6 (2015) com sintaxe de classes
  • Vantagens:
    • Organização: Estrutura mais lógica e intuitiva
    • Reutilização: Reaproveitamento de código
    • Manutenção: Código mais fácil de manter
    • Escalabilidade: Sistemas complexos mais gerenciáveis
"A POO nos permite modelar o mundo real em nosso código, criando sistemas mais intuitivos e organizados."

Conceitos do Paradigma da Programação Orientada a Objetos

A Programação Orientada a Objetos se baseia em quatro pilares fundamentais:

Abstração

Identificar características essenciais, ignorando detalhes menos importantes.

Encapsulamento

Esconder detalhes internos e expor apenas o necessário, protegendo os dados.

Herança

Mecanismo que permite que uma classe herde características de outra.

Polimorfismo

Capacidade de objetos de classes diferentes responderem à mesma mensagem de maneiras diferentes.

"Estes quatro pilares são a base de qualquer sistema orientado a objetos bem projetado."

Objetos e Classes em JavaScript

Em JavaScript, objetos são coleções de propriedades e métodos:

Objetos Literais

// Criando um objeto literal const pessoa = { nome: "Ana", idade: 30, saudacao: function() { return `Olá, meu nome é ${this.nome}`; } }; console.log(pessoa.saudacao()); // "Olá, meu nome é Ana"

Classes (ES6+)

// Definindo uma classe class Pessoa { constructor(nome, idade) { this.nome = nome; this.idade = idade; } saudacao() { return `Olá, meu nome é ${this.nome}`; } } // Criando objetos a partir da classe const pessoa1 = new Pessoa("Ana", 30); console.log(pessoa1.saudacao()); // "Olá, meu nome é Ana"

Atributos e Métodos

Classes em JavaScript podem ter diferentes tipos de atributos e métodos:

Atributos de Instância

class Pessoa { constructor(nome, idade) { this.nome = nome; // Atributo público this.idade = idade; // Atributo público this._cpf = null; // Convenção para "privado" this.#senha = "1234"; // Realmente privado (ES2022+) } }

Métodos de Instância

class Pessoa { constructor(nome, idade) { this.nome = nome; this.idade = idade; } // Método público apresentar() { return `Olá, meu nome é ${this.nome}`; } // Método privado (ES2022+) #validarIdade() { return this.idade >= 18; } }

Atributos e Métodos Estáticos

Atributos e métodos estáticos pertencem à classe, não às instâncias:

Atributos Estáticos

class Matematica { static PI = 3.14159; // Atributo estático static calcularAreaCirculo(raio) { return Matematica.PI * raio * raio; } } console.log(Matematica.PI); // 3.14159

Métodos Estáticos

class Utilitarios { static gerarIdAleatorio() { return Math.floor(Math.random() * 1000); } static formatarData(data) { return data.toLocaleDateString('pt-BR'); } } const id = Utilitarios.gerarIdAleatorio(); console.log(id); // Número aleatório entre 0 e 999

Getters e Setters

Getters e setters permitem controlar o acesso às propriedades de um objeto:

Definindo Getters e Setters

class Conta { constructor(titular) { this._titular = titular; this._saldo = 0; } // Getter get saldo() { return this._saldo; } // Setter set saldo(valor) { if (valor < 0) { console.error("Saldo não pode ser negativo"); return; } this._saldo = valor; } } const minhaConta = new Conta("João"); minhaConta.saldo = 1000; // Usa o setter console.log(minhaConta.saldo); // Usa o getter, exibe 1000 minhaConta.saldo = -500; // Erro: "Saldo não pode ser negativo"

Herança

A herança permite que uma classe herde propriedades e métodos de outra:

Implementando Herança

// Classe pai (superclasse) class Animal { constructor(nome) { this.nome = nome; } fazerSom() { return "Som genérico de animal"; } } // Classe filha (subclasse) class Cachorro extends Animal { constructor(nome, raca) { super(nome); // Chama o construtor da classe pai this.raca = raca; } fazerSom() { return "Au au!"; // Sobrescreve o método da classe pai } abanarRabo() { return `${this.nome} está abanando o rabo!`; } } const rex = new Cachorro("Rex", "Labrador"); console.log(rex.nome); // "Rex" console.log(rex.fazerSom()); // "Au au!" console.log(rex.abanarRabo()); // "Rex está abanando o rabo!"

Polimorfismo

O polimorfismo permite que objetos de diferentes classes sejam tratados de maneira uniforme:

Exemplo de Polimorfismo

class Animal { fazerSom() { return "Som genérico"; } } class Cachorro extends Animal { fazerSom() { return "Au au!"; } } class Gato extends Animal { fazerSom() { return "Miau!"; } } // Função que trabalha com qualquer Animal function ouvirAnimal(animal) { console.log(animal.fazerSom()); } const rex = new Cachorro(); const felix = new Gato(); ouvirAnimal(rex); // "Au au!" ouvirAnimal(felix); // "Miau!"

No exemplo acima, a função ouvirAnimal() funciona com qualquer objeto que seja um Animal ou derivado de Animal, demonstrando o polimorfismo em ação.

Abstração e Classes Abstratas

A abstração permite focar nas características essenciais, ignorando detalhes menos importantes:

Simulando Classes Abstratas em JavaScript

JavaScript não tem suporte nativo para classes abstratas, mas podemos simular esse comportamento:

class FormaGeometrica { constructor() { if (this.constructor === FormaGeometrica) { throw new Error("Classe abstrata não pode ser instanciada"); } } calcularArea() { throw new Error("Método abstrato deve ser implementado"); } } class Retangulo extends FormaGeometrica { constructor(largura, altura) { super(); this.largura = largura; this.altura = altura; } calcularArea() { return this.largura * this.altura; } } // const forma = new FormaGeometrica(); // Erro: "Classe abstrata não pode ser instanciada" const retangulo = new Retangulo(5, 3); console.log(retangulo.calcularArea()); // 15

Interfaces em JavaScript

JavaScript não tem suporte nativo para interfaces, mas podemos simular esse conceito:

Simulando Interfaces

// Definindo uma "interface" const VeiculoInterface = { acelerar() {}, frear() {}, virar(direcao) {} }; // Implementando a "interface" class Carro { constructor(marca, modelo) { this.marca = marca; this.modelo = modelo; } acelerar() { return `${this.marca} ${this.modelo} está acelerando`; } frear() { return `${this.marca} ${this.modelo} está freando`; } virar(direcao) { return `${this.marca} ${this.modelo} está virando para ${direcao}`; } } // Verificando se a classe implementa a interface function verificarInterface(obj, interfaceObj) { return Object.keys(interfaceObj).every( metodo => typeof obj[metodo] === 'function' ); } const meuCarro = new Carro("Toyota", "Corolla"); console.log(verificarInterface(meuCarro, VeiculoInterface)); // true

Com TypeScript, podemos usar interfaces de forma mais robusta e com verificação em tempo de compilação.

Agregação e Composição

Agregação e composição são formas de estabelecer relacionamentos entre objetos:

Agregação

Relação "tem um" onde os objetos podem existir independentemente.

// Agregação class Departamento { constructor(nome) { this.nome = nome; this.funcionarios = []; } adicionarFuncionario(funcionario) { this.funcionarios.push(funcionario); } } class Funcionario { constructor(nome) { this.nome = nome; } } const rh = new Departamento("RH"); const joao = new Funcionario("João"); rh.adicionarFuncionario(joao); // O funcionário existe mesmo // se o departamento for removido

Composição

Relação "parte de" onde os objetos dependem do objeto principal.

// Composição class Carro { constructor(modelo) { this.modelo = modelo; this.motor = new Motor(2.0); this.rodas = [ new Roda(), new Roda(), new Roda(), new Roda() ]; } } class Motor { constructor(potencia) { this.potencia = potencia; } } class Roda { constructor() { this.aro = 17; } } const meuCarro = new Carro("Sedan"); // Se o carro for destruído, // motor e rodas também são

Encapsulamento

O encapsulamento protege os dados internos de um objeto, controlando o acesso a eles:

Técnicas de Encapsulamento em JavaScript

1. Convenção de Nomenclatura

class Pessoa { constructor(nome, idade) { this.nome = nome; // Público this._idade = idade; // "Privado" por convenção } getIdade() { return this._idade; } setIdade(valor) { if (valor >= 0) { this._idade = valor; } } }

2. Campos Privados (ES2022+)

class ContaBancaria { #saldo = 0; // Campo realmente privado constructor(titular) { this.titular = titular; } depositar(valor) { if (valor > 0) { this.#saldo += valor; return true; } return false; } consultarSaldo() { return this.#saldo; } } const conta = new ContaBancaria("Maria"); conta.depositar(1000); console.log(conta.consultarSaldo()); // 1000 // console.log(conta.#saldo); // Erro: campo privado

Tratamento de Exceções

O tratamento de exceções permite lidar com erros de forma elegante em aplicações orientadas a objetos:

Try-Catch-Finally

class ContaBancaria { constructor(saldoInicial = 0) { if (saldoInicial < 0) { throw new Error("Saldo inicial não pode ser negativo"); } this.saldo = saldoInicial; } sacar(valor) { if (valor <= 0) { throw new Error("Valor de saque deve ser positivo"); } if (valor > this.saldo) { throw new Error("Saldo insuficiente"); } this.saldo -= valor; return valor; } } try { const conta = new ContaBancaria(1000); conta.sacar(500); console.log(`Saldo atual: ${conta.saldo}`); // Tentativa de saque inválido conta.sacar(2000); } catch (erro) { console.error(`Erro: ${erro.message}`); } finally { console.log("Operação bancária finalizada"); }

Criando Exceções Personalizadas

class SaldoInsuficienteError extends Error { constructor(saldoAtual, valorSaque) { super(`Saldo insuficiente: ${saldoAtual} disponível, ${valorSaque} solicitado`); this.name = "SaldoInsuficienteError"; this.saldoAtual = saldoAtual; this.valorSaque = valorSaque; } }
