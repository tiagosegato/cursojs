Introdução à Web e Suas Tecnologias

A World Wide Web é um sistema complexo construído sobre algumas tecnologias fundamentais:

HTML (HyperText Markup Language)

A espinha dorsal da web. Define a estrutura e o conteúdo das páginas.

<h1>Olá, Mundo!</h1> <p>Este é um parágrafo.</p>

CSS (Cascading Style Sheets)

Responsável pela apresentação e estilo visual das páginas HTML.

h1 { color: blue; font-size: 24px; }

JavaScript (JS)

Torna as páginas interativas e dinâmicas. É a linguagem de programação do navegador.

document.getElementById("meuBotao") .addEventListener("click", function() { alert("Botão clicado!"); });

Como funciona?

O navegador requisita uma página (HTML), que pode referenciar arquivos CSS e JS. O navegador então interpreta esses arquivos para renderizar a página visual e interativa para o usuário.

Papel do Desenvolvedor Front-end e Back-end

No desenvolvimento web, as responsabilidades são frequentemente divididas entre Front-end e Back-end:

Desenvolvedor Front-end

Focado na parte da aplicação com a qual o usuário interage diretamente (o "cliente" ou navegador).

  • Tecnologias: HTML, CSS, JavaScript (e frameworks como React, Angular, Vue.js).
  • Responsabilidades: Interface do usuário (UI), experiência do usuário (UX), responsividade, interatividade.
  • Objetivo: Criar interfaces atraentes, intuitivas e eficientes.

Desenvolvedor Back-end

Trabalha nos "bastidores" da aplicação, lidando com o servidor, banco de dados e lógica de negócios.

  • Tecnologias: Node.js (com JavaScript), Python, Java, Ruby, PHP, bancos de dados (SQL, NoSQL).
  • Responsabilidades: APIs, lógica de servidor, gerenciamento de dados, segurança, performance.
  • Objetivo: Garantir que a aplicação funcione corretamente, seja segura e escalável.

Desenvolvedor Full-stack: Profissional que possui habilidades tanto em Front-end quanto em Back-end.

Algoritmo e Suas Representações

Um algoritmo é uma sequência finita de passos bem definidos para resolver um problema ou realizar uma tarefa.

Representações de Algoritmos:

  • Descrição Narrativa: Usando linguagem natural para descrever os passos. Ex: "Para somar dois números, pegue o primeiro número, adicione o segundo número a ele, e o resultado é a soma."
  • Fluxograma: Representação gráfica usando símbolos padronizados para ilustrar o fluxo de controle do algoritmo.
  • Pseudocódigo: Uma linguagem informal, semelhante a uma linguagem de programação, mas mais flexível e focada na lógica. Ex:
    INÍCIO LER numero1 LER numero2 soma = numero1 + numero2 ESCREVER soma FIM

Entender algoritmos é crucial, pois eles são a base de qualquer programa de computador.

Ferramentas e Ambientes de Desenvolvimento

Para desenvolver com JavaScript, algumas ferramentas são essenciais:

  • Navegador Web: O ambiente principal onde o JavaScript do lado do cliente é executado. Ferramentas de desenvolvedor embutidas (Console, Debugger, Inspector) são cruciais. (Ex: Chrome, Firefox, Edge).
  • Editor de Código ou IDE (Ambiente de Desenvolvimento Integrado): Programas para escrever, editar e gerenciar código. (Ex: VS Code, Sublime Text, WebStorm).
  • Node.js: Permite executar JavaScript fora do navegador, no lado do servidor. Útil para ferramentas de build, automação e desenvolvimento back-end.
  • Controle de Versão (Git): Ferramenta para rastrear mudanças no código, colaborar com outros desenvolvedores e gerenciar diferentes versões do projeto. (Plataformas como GitHub, GitLab, Bitbucket).

Configurar um bom ambiente de desenvolvimento aumenta a produtividade e facilita o processo de criação de software.

Definição de Memória

Em computação, memória refere-se à capacidade de um sistema de armazenar dados e instruções, temporária ou permanentemente.

Tipos de Memória (Conceitual):

  • Memória Principal (RAM - Random Access Memory): Memória volátil usada pelo processador para armazenar dados e programas em execução. É rápida, mas perde seu conteúdo quando a energia é desligada.
  • Memória Secundária (Armazenamento): Memória não volátil usada para armazenamento de longo prazo de arquivos e programas (Ex: HDs, SSDs, pen drives).

Gerenciamento de Memória em JavaScript:

JavaScript possui um sistema de gerenciamento automático de memória, incluindo um "coletor de lixo" (garbage collector).

  • Alocação: Quando você cria variáveis, objetos ou arrays, o JavaScript aloca espaço na memória para eles.
  • Coleta de Lixo: O coletor de lixo periodicamente identifica e libera memória que não está mais sendo usada pelo programa, prevenindo vazamentos de memória.

Embora automático, entender como a memória funciona ajuda a escrever código mais eficiente.

Definição de Variáveis

Variáveis são "contêineres" nomeados usados para armazenar dados na memória que podem ser referenciados e manipulados em um programa.

Declarando Variáveis em JavaScript:

  • var (Legado): Forma mais antiga de declarar variáveis. Possui escopo de função ou global. Evite em código moderno.
    var nome = "João";
  • let (Moderno): Permite declarar variáveis com escopo de bloco. O valor pode ser reatribuído.
    let idade = 30; idade = 31; // Válido
  • const (Moderno): Permite declarar constantes com escopo de bloco. O valor não pode ser reatribuído após a inicialização (para tipos primitivos; para objetos, a referência é constante, mas o conteúdo pode mudar).
    const PI = 3.14159; // PI = 3.14; // Erro! const pessoa = { nome: "Ana" }; pessoa.nome = "Maria"; // Válido

Escolher entre let e const depende se o valor da variável precisará mudar.

Conceitos Básicos de Lógica de Programação

Lógica de programação é a organização coerente de instruções para que um programa execute uma tarefa específica.

Elementos Fundamentais:

  • Sequência: Instruções são executadas na ordem em que aparecem.
  • Decisão (Condicionais): Permitem que o programa escolha diferentes caminhos de execução com base em condições (Ex: if, else if, else, switch).
  • Repetição (Loops): Permitem que um bloco de código seja executado múltiplas vezes (Ex: for, while, do...while).
  • Entrada e Saída de Dados: Como o programa recebe informações (Ex: do usuário, de um arquivo) e como ele apresenta resultados (Ex: na tela, em um arquivo).
  • Operadores: Símbolos que realizam operações sobre valores (Ex: aritméticos +, -; lógicos &&, ||; comparação >, ===).
  • Tipos de Dados: Classificação dos valores que uma variável pode armazenar (Ex: números, strings, booleanos).

Dominar esses conceitos é essencial para construir qualquer tipo de software.

Estruturas de Controle (Condicionais)

Estruturas condicionais permitem que seu programa tome decisões e execute diferentes blocos de código com base em certas condições serem verdadeiras ou falsas.

if, else if, else:

let temperatura = 25; if (temperatura > 30) { console.log("Está muito quente!"); } else if (temperatura > 20) { console.log("Temperatura agradável."); } else { console.log("Está frio!"); }

switch:

Útil para múltiplas verificações contra um único valor.

let diaDaSemana = "segunda"; switch (diaDaSemana) { case "sábado": case "domingo": console.log("Fim de semana!"); break; case "segunda": console.log("Início da semana de trabalho."); break; default: console.log("Dia útil."); }

Estruturas de Controle (Loops)

Estruturas de repetição (loops) permitem executar um bloco de código várias vezes.

for loop:

Ideal quando o número de iterações é conhecido.

for (let i = 0; i < 5; i++) { console.log("Iteração número: " + i); }

while loop:

Executa enquanto uma condição for verdadeira. A condição é verificada antes da execução do bloco.

let contador = 0; while (contador < 3) { console.log("Contador (while): " + contador); contador++; }

do...while loop:

Similar ao while, mas o bloco de código é executado pelo menos uma vez, pois a condição é verificada após a execução.

let j = 0; do { console.log("Contador (do...while): " + j); j++; } while (j < 3);

Funções e Modularização

Funções são blocos de código reutilizáveis que realizam uma tarefa específica. Modularização é a prática de dividir um programa em partes menores e independentes (módulos ou funções).

Declarando Funções:

// Declaração de função function somar(a, b) { return a + b; } // Chamando a função let resultado = somar(5, 3); console.log(resultado); // 8

Arrow Functions (ES6+):

Uma sintaxe mais concisa para criar funções.

const multiplicar = (x, y) => { return x * y; }; // Ou ainda mais conciso para retornos simples const subtrair = (a, b) => a - b; console.log(multiplicar(4, 2)); // 8 console.log(subtrair(10, 7)); // 3

Benefícios da Modularização:

  • Reutilização de Código: Evita repetição.
  • Legibilidade: Código mais fácil de entender.
  • Manutenção: Alterações em uma parte não afetam outras drasticamente.
  • Testabilidade: Módulos menores são mais fáceis de testar.
