1.1. Introdução ao Node.js

Até pouco tempo atrás, o JavaScript vivia exclusivamente dentro dos navegadores de internet (como Chrome, Firefox) para criar interatividade em páginas web (o frontend).

O que é Node.js?

  • É um ambiente de execução que permite rodar código JavaScript no lado do servidor (no backend).
  • Não é uma nova linguagem de programação. É o mesmo JavaScript que você já conhece.
  • Não é um framework. É a plataforma sobre a qual construiremos nossas aplicações.
  • É rápido e eficiente. Foi construído sobre o motor V8 do Google Chrome.
  • Isso permitiu que desenvolvedores usassem uma única linguagem (JavaScript) tanto para o frontend quanto para o backend, simplificando o desenvolvimento.

1.3. Frontend, Backend e APIs

Imagine que uma aplicação web é um restaurante.

  • Frontend (O Salão): É a parte com a qual o usuário interage diretamente (telas, botões, formulários). É o que você vê. Tecnologias: HTML, CSS, JavaScript (React, Vue, Angular).
  • Backend (A Cozinha): É a parte que funciona nos "bastidores". Processa regras de negócio, acessa o banco de dados e cuida da segurança. É o cérebro da operação. Tecnologias: Node.js, Python, Java.
  • API (O Garçom): É o intermediário que permite a comunicação entre o Frontend e o Backend. A API (Application Programming Interface) é um conjunto de regras e protocolos que define como as duas partes devem se comunicar.

1.4. Framework Express.js

Node.js puro é muito "cru". Seria como construir um carro do zero, fabricando cada parafuso.

O que é o Express.js?

É um framework web minimalista e flexível para Node.js. Ele nos fornece um conjunto de ferramentas para construir APIs de forma muito mais rápida e organizada, como gerenciamento de rotas e de requisições/respostas.

1.5. Roteamento e Endpoints

Uma rota é um "caminho" em nosso servidor que executa uma função específica. A combinação do método HTTP e do caminho forma o que chamamos de endpoint.

Métodos HTTP (Verbos):

  • GET: Para buscar dados.
  • POST: Para criar um novo dado.
  • PUT / PATCH: Para atualizar um dado existente.
  • DELETE: Para remover um dado.

Caminho (Path):

O endereço do recurso (ex: /tarefas, /usuarios).

2.3. Criando o Servidor Básico

Crie um arquivo index.js com o seguinte código:

const express = require('express');
const app = express();
const PORT = 3000;

// Middleware para o Express entender JSON
app.use(express.json());

// Nosso "banco de dados" em memória
let tarefas = [
  { id: 1, titulo: "Aprender Node.js", concluida: false },
  { id: 2, titulo: "Criar projeto com Express", concluida: true },
];

app.get('/', (req, res) => {
  res.send('<h1>API de Tarefas está funcionando!</h1>');
});

// Implementação do CRUD virá aqui...

app.listen(PORT, () => {
  console.log(`Servidor rodando na porta http://localhost:${PORT}`);
});

Execute com node index.js.

2.4. Implementando o CRUD

Adicione as seguintes rotas ao seu index.js:

// Listar todas as tarefas (GET /tarefas)
app.get('/tarefas', (req, res) => {
  res.json(tarefas);
});

// Obter uma tarefa por ID (GET /tarefas/:id)
app.get('/tarefas/:id', (req, res) => {
  const tarefa = tarefas.find(t => t.id === parseInt(req.params.id));
  if (!tarefa) return res.status(404).json({ mensagem: "Tarefa não encontrada!" });
  res.json(tarefa);
});

2.4. Implementando o CRUD

Continuando as rotas CRUD:

// Criar uma nova tarefa (POST /tarefas)
app.post('/tarefas', (req, res) => {
  const novaTarefa = {
    id: tarefas.length > 0 ? tarefas[tarefas.length - 1].id + 1 : 1,
    titulo: req.body.titulo,
    concluida: false
  };
  tarefas.push(novaTarefa);
  res.status(201).json(novaTarefa);
});

// Atualizar uma tarefa (PUT /tarefas/:id)
app.put('/tarefas/:id', (req, res) => {
  const indice = tarefas.findIndex(t => t.id === parseInt(req.params.id));
  if (indice === -1) return res.status(404).json({ mensagem: "Tarefa não encontrada!" });
  
  tarefas[indice].titulo = req.body.titulo || tarefas[indice].titulo;
  tarefas[indice].concluida = typeof req.body.concluida === 'boolean' ? req.body.concluida : tarefas[indice].concluida;
  
  res.json(tarefas[indice]);
});

// Deletar uma tarefa (DELETE /tarefas/:id)
app.delete('/tarefas/:id', (req, res) => {
  const indice = tarefas.findIndex(t => t.id === parseInt(req.params.id));

Módulo 1: Persistência de Dados

Operações de banco de dados são assíncronas. O Node.js não espera por elas.

Programação Assíncrona:

  • Callbacks: O jeito antigo, leva ao "Callback Hell".
  • Promises: Uma "promessa" de um resultado futuro, melhora a legibilidade com .then() e .catch().
  • Async/Await: A forma mais moderna. Escrevemos código assíncrono que parece síncrono. Usaremos este método.

Programa de Formação Acelerada

Integrando o Sequelize com SQLite

1. Instale as dependências:

npm install sequelize sqlite3
npm install --save-dev sequelize-cli

2. Inicialize o Sequelize:

npx sequelize-cli init

3. Configure a conexão (config/config.json):

{
  "development": {
    "dialect": "sqlite",
    "storage": "database.sqlite"
  }
}

4. Crie o Modelo e a Migration:

npx sequelize-cli model:generate --name Tarefa --attributes titulo:string,concluida:boolean

5. Execute a Migration para criar a tabela:

npx sequelize-cli db:migrate

Refatorando o CRUD

Substitua as funções das rotas pelas versões async que usam os métodos do Sequelize.

// index.js
const { Tarefa } = require('./models');

// GET /tarefas
app.get('/tarefas', async (req, res) => {
  const tarefas = await Tarefa.findAll();
  res.json(tarefas);
});

// POST /tarefas
app.post('/tarefas', async (req, res) => {
  const novaTarefa = await Tarefa.create(req.body);
  res.status(201).json(novaTarefa);
});

// E assim por diante para PUT e DELETE...

Criando um Middleware de Log

const logMiddleware = (req, res, next) => {

Tratamento de Erros

1. Instale express-async-errors:

npm install express-async-errors

2. Importe no topo do index.js:

require('express-async-errors');

3. Adicione este middleware no final do arquivo, antes de app.listen:

app.use((error, req, res, next) => {
  console.error(error.stack);
  res.status(500).json({ mensagem: "Ocorreu um erro interno no servidor." });
});

Programa de Formação Acelerada

Rotas de Registro e Login

  • Registro (POST /auth/registrar): Recebe email e senha, criptografa a senha com bcrypt.hash() e salva o novo usuário.
  • Login (POST /auth/login): Recebe email e senha, busca o usuário, compara a senha com bcrypt.compare(). Se for válida, gera um token com jwt.sign().
25
Programa de Formação Acelerada

Middleware de Autenticação

Crie um Middleware de Autenticação (authMiddleware.js):

  • Ele verifica se o header Authorization existe e contém um token válido.
  • Usa jwt.verify() para validar o token.
  • Se for válido, anexa o ID do usuário ao objeto req (req.usuarioId = decoded.id;) e chama next().
  • Se não, retorna um erro 401 (Não Autorizado).

Variáveis de Ambiente (.env)

Nunca coloque dados sensíveis (senhas, segredos JWT) no código.

  1. Instale dotenv: npm install dotenv.
  2. Crie um arquivo .env na raiz do projeto: 
    PORT=3001
JWT_SECRET=meu_segredo_super_secreto
  3. Adicione .env ao seu arquivo .gitignore.
  4. Carregue as variáveis no topo do index.js: require('dotenv').config();.
  5. Use-as com process.env.NOME_DA_VARIAVEL.

SSL e Deploy em VPS

  • SSL: Criptografa a comunicação. Essencial. Em produção, geralmente é gerenciado por um Reverse Proxy como o Nginx.
  • Deploy em VPS (Visão Geral):
    1. Configurar Servidor: Instale Node.js, Git, Nginx.
    2. Transferir Código: Use Git para clonar seu projeto.
    3. Gerenciador de Processos: Use o PM2 (npm install -g pm2) para manter sua aplicação rodando 24/7 (pm2 start index.js).
    4. Configurar Reverse Proxy (Nginx): Configure o Nginx para direcionar o tráfego da internet (porta 80/443) para a porta da sua aplicação Node (ex: 3000).
