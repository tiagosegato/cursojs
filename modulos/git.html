Módulo 2: Git, GitHub e GitLab

Controle de versão, colaboração em equipe e gerenciamento de código-fonte

Visão Geral do Módulo

Neste módulo, você aprenderá sobre sistemas de controle de versão, com foco no Git e nas plataformas GitHub e GitLab. Essas ferramentas são essenciais para o desenvolvimento de software moderno, permitindo o trabalho colaborativo, o controle de alterações no código e a gestão eficiente de projetos.

Carga Horária

3 horas

Conteúdo Programático

1. Introdução ao Controle de Versão
  • O que é controle de versão e por que é importante
  • Sistemas de controle de versão centralizados vs. distribuídos
  • História e evolução do Git

2. Configuração e Conceitos Básicos do Git

  • Adicionando arquivos ao staging area (git add)
  • Criando commits (git commit)

  • Criando uma conta no GitHub
  • Criando e configurando repositórios

  • GitHub Pages: hospedagem de sites estáticos

  • Diferenças entre GitHub e GitLab
  • Criando uma conta no GitLab
  • Criando e configurando projetos
  • GitLab CI/CD: pipelines de integração contínua
  • Merge Requests no GitLab
  • GitLab Pages
8. Boas Práticas e Fluxos de Trabalho
  • Escrevendo boas mensagens de commit
  • Gitflow: um modelo de branching para equipes
  • GitHub Flow: um fluxo de trabalho simplificado
  • Conventional Commits: padronização de mensagens
  • Gerenciamento de versões com tags
  • Ignorando arquivos com .gitignore

Exemplos Práticos

Exemplo 1: Configuração Inicial do Git

# Configurando nome de usuário e email
git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"

# Verificando as configurações
git config --list

# Configurando o editor padrão
git config --global core.editor "code --wait"  # Para VS Code

# Configurando aliases úteis
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit

Exemplo 2: Criando um Repositório para o Sistema de Vendas

# Criando um novo diretório para o projeto
mkdir sistema-vendas
cd sistema-vendas

# Inicializando o repositório Git
git init

# Criando um arquivo README.md
echo "# Sistema de Vendas" > README.md
echo "" >> README.md
echo "Sistema de gerenciamento de vendas desenvolvido com Node.js e Express." >> README.md

# Adicionando o arquivo ao staging area
git add README.md

# Criando o primeiro commit
git commit -m "Inicialização do projeto: Sistema de Vendas"

# Criando um arquivo para o modelo de Produto
mkdir models
echo "// Modelo de Produto para o Sistema de Vendas" > models/Produto.js
echo "" >> models/Produto.js
echo "class Produto {" >> models/Produto.js
echo "  constructor(codigo, unidade, descricao, valor_unitario) {" >> models/Produto.js
echo "    this.codigo = codigo;" >> models/Produto.js
echo "    this.unidade = unidade;" >> models/Produto.js
echo "    this.descricao = descricao;" >> models/Produto.js
echo "    this.valor_unitario = valor_unitario;" >> models/Produto.js
echo "  }" >> models/Produto.js
echo "}" >> models/Produto.js
echo "" >> models/Produto.js
echo "module.exports = Produto;" >> models/Produto.js

# Adicionando o arquivo de modelo ao staging area
git add models/Produto.js

# Criando um novo commit
git commit -m "Adiciona modelo de Produto"

# Verificando o histórico de commits
git log --oneline

Exemplo 3: Trabalhando com Branches no Projeto de Vendas

# Criando uma nova branch para desenvolver a funcionalidade de listagem de produtos
git checkout -b feature/listagem-produtos

# Criando um arquivo para o controlador de produtos
mkdir controllers
echo "// Controlador de Produtos" > controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "const Produto = require('../models/Produto');" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "class ProdutoController {" >> controllers/ProdutoController.js
echo "  constructor() {" >> controllers/ProdutoController.js
echo "    this.produtos = [" >> controllers/ProdutoController.js
echo "      new Produto(25, 'Kg', 'Queijo', 0.97)," >> controllers/ProdutoController.js
echo "      new Produto(31, 'BAR', 'Chocolate', 0.87)," >> controllers/ProdutoController.js
echo "      new Produto(78, 'L', 'Vinho', 2.00)," >> controllers/ProdutoController.js
echo "      new Produto(22, 'M', 'Linho', 0.11)," >> controllers/ProdutoController.js
echo "      new Produto(30, 'SAC', 'Açúcar', 0.30)" >> controllers/ProdutoController.js
echo "    ];" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "  listarTodos() {" >> controllers/ProdutoController.js
echo "    return this.produtos;" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "}" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "module.exports = ProdutoController;" >> controllers/ProdutoController.js

# Adicionando o arquivo ao staging area
git add controllers/ProdutoController.js

# Criando um commit na branch feature
git commit -m "Adiciona controlador de produtos com método de listagem"

# Voltando para a branch principal
git checkout main

# Criando outra branch para desenvolver a funcionalidade de busca de produtos
git checkout -b feature/busca-produtos

# Modificando o controlador para adicionar a funcionalidade de busca
echo "// Controlador de Produtos" > controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "const Produto = require('../models/Produto');" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "class ProdutoController {" >> controllers/ProdutoController.js
echo "  constructor() {" >> controllers/ProdutoController.js
echo "    this.produtos = [" >> controllers/ProdutoController.js
echo "      new Produto(25, 'Kg', 'Queijo', 0.97)," >> controllers/ProdutoController.js
echo "      new Produto(31, 'BAR', 'Chocolate', 0.87)," >> controllers/ProdutoController.js
echo "      new Produto(78, 'L', 'Vinho', 2.00)," >> controllers/ProdutoController.js
echo "      new Produto(22, 'M', 'Linho', 0.11)," >> controllers/ProdutoController.js
echo "      new Produto(30, 'SAC', 'Açúcar', 0.30)" >> controllers/ProdutoController.js
echo "    ];" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "  listarTodos() {" >> controllers/ProdutoController.js
echo "    return this.produtos;" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "  buscarPorCodigo(codigo) {" >> controllers/ProdutoController.js
echo "    return this.produtos.find(produto => produto.codigo === codigo);" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "  buscarPorDescricao(termo) {" >> controllers/ProdutoController.js
echo "    return this.produtos.filter(produto => " >> controllers/ProdutoController.js
echo "      produto.descricao.toLowerCase().includes(termo.toLowerCase())" >> controllers/ProdutoController.js
echo "    );" >> controllers/ProdutoController.js
echo "  }" >> controllers/ProdutoController.js
echo "}" >> controllers/ProdutoController.js
echo "" >> controllers/ProdutoController.js
echo "module.exports = ProdutoController;" >> controllers/ProdutoController.js

# Adicionando as alterações ao staging area
git add controllers/ProdutoController.js

# Criando um commit na branch de busca
git commit -m "Adiciona métodos de busca por código e descrição"

# Voltando para a branch principal
git checkout main

# Mesclando a branch de listagem
git merge feature/listagem-produtos

# Mesclando a branch de busca (pode gerar conflitos que precisarão ser resolvidos)
git merge feature/busca-produtos

Exemplo 4: Configurando um Repositório Remoto no GitHub

# Criando um repositório no GitHub (via interface web)
# Após criar o repositório, conecte seu repositório local ao remoto

# Adicionando o repositório remoto
git remote add origin https://github.com/seu-usuario/sistema-vendas.git

# Verificando os repositórios remotos configurados
git remote -v

# Enviando o código para o repositório remoto
git push -u origin main

# Criando um arquivo .gitignore para ignorar arquivos desnecessários
echo "# Arquivos e diretórios a serem ignorados pelo Git" > .gitignore
echo "" >> .gitignore
echo "# Dependências" >> .gitignore
echo "node_modules/" >> .gitignore
echo "package-lock.json" >> .gitignore
echo "" >> .gitignore
echo "# Arquivos de ambiente" >> .gitignore
echo ".env" >> .gitignore
echo ".env.local" >> .gitignore
echo "" >> .gitignore
echo "# Logs" >> .gitignore
echo "logs/" >> .gitignore
echo "*.log" >> .gitignore
echo "" >> .gitignore
echo "# Diretório de build" >> .gitignore
echo "dist/" >> .gitignore
echo "build/" >> .gitignore
echo "" >> .gitignore
echo "# Arquivos do sistema" >> .gitignore
echo ".DS_Store" >> .gitignore
echo "Thumbs.db" >> .gitignore

# Adicionando o .gitignore ao staging area
git add .gitignore

# Criando um commit
git commit -m "Adiciona arquivo .gitignore"

# Enviando as alterações para o repositório remoto
git push origin main

Exercícios Práticos

Para fixar os conceitos aprendidos neste módulo, recomendamos a realização dos seguintes exercícios:

  1. Configure o Git em sua máquina local com seu nome e email.
  2. Crie um repositório local para o projeto do Sistema de Vendas.
  3. Adicione arquivos iniciais ao repositório e faça commits com mensagens descritivas.
  4. Crie branches para desenvolver diferentes funcionalidades do sistema.
  5. Pratique a mesclagem de branches e resolução de conflitos.
  6. Crie uma conta no GitHub (se ainda não tiver) e crie um repositório remoto.
  7. Conecte seu repositório local ao remoto e envie seu código.
  8. Crie um arquivo README.md detalhado para o projeto.
  9. Configure um arquivo .gitignore adequado para projetos Node.js.
  10. Simule um fluxo de trabalho colaborativo com um colega, utilizando pull requests.

Você encontrará mais exercícios e desafios na seção de Exercícios.
